第二章　方法

第一節　設計原則

實現繁體中文豎排 EPUB 需要遵循幾項核心設計原則。這些原則既能確保在現有閱讀器上的兼容性，又為未來的標準化預留擴展空間。

一、書寫模式分層設定

建議在 html 和 body 元素上同時設定書寫模式，確保雙重保險。範例如下：

html  { writing-mode: vertical-rl; }
body  { writing-mode: vertical-rl; }

二、Vendor Prefixes 完整性

EPUB 閱讀器市場高度碎片化，必須同時提供多個前綴才能覆蓋絕大多數設備：

  • -epub-writing-mode — EPUB 標準前綴
  • -webkit-writing-mode — Apple Books / Google Play Books
  • -ms-writing-mode — IE / 舊版 Edge
  • writing-mode — 現代標準屬性

三、翻頁方向一致性

在 content.opf 的 <spine> 中必須設定：

page-progression-direction="rtl"

此屬性使閱讀器的頁面流向與豎排文字方向一致，確保從右至左翻頁的正確體驗。

第二節　實作細節

一、字體選擇

推薦使用以下字體系列，優先嵌入 Noto Serif CJK 或思源宋體：

  • Noto Serif CJK TC — Google / Adobe 合作開發
  • 思源宋體（Source Han Serif） — Adobe 開放字體
  • 標楷體（DFKai-SB） — 臺灣教育部門標準
  • 明體（MingLiU） — 香港常用

二、嵌入字體步驟

在 EPUB 中嵌入中文字體需要三個步驟：

  1. 在 content.opf 的 <manifest> 中聲明字體檔案
  2. 在 CSS 中使用 @font-face 規則引用
  3. 將字體檔案放入 OEBPS/fonts/ 目錄

三、Tatechuyoko（縱中橫）

對於阿拉伯數字、英文單詞或短拉丁文字，可以使用 text-combine-upright 屬性讓它們在豎排中保持橫排：

四、分頁控制

使用 page-break-before、page-break-after 和 break-inside 避免章節標題被截斷，以及段落在中間分頁：

h1, h2 { page-break-before: avoid; break-before: avoid; }
p       { page-break-inside: avoid; break-inside: avoid; }

五、Ruby 注音

繁體中文常用 <ruby> 元素標註注音或振假名：

範例：孔子ㄎㄨㄥˇ ㄗˇ、詩經ㄕ ㄐㄧㄥ

預設注音顯示在文字上方（ruby-position: over），如需顯示在下方，可改為 under。

六、封面設計

建議使用 SVG 格式製作封面，以確保在不同螢幕尺寸和解析度上的一致顯示。本範本中的封面即採用 SVG 格式，包含書名裝飾線和背景。
