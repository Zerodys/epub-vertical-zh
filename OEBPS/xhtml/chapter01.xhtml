第一章　概述

第一節　背景介紹

繁體中文的豎排書寫有著數千年的歷史傳統，是中華文化重要的視覺表現形式。與橫排由左至右不同，傳統中文書籍採用由右至左、由上至下的閱讀順序，這種設計不僅符合人眼的自然掃描路徑，也造就了書法、版刻等獨特的藝術形式。

在數位出版的時代，如何在電子閱讀器上忠實呈現傳統豎排文字，成為技術上的一大挑戰。本範本整合了 EPUB 3 標準與 CSS Writing Modes 規範，提供一套完整可行的解決方案。

「書法之道，在乎氣韻生動。橫豎撇捺，皆有法度。」

本 EPUB 範本支援以下核心功能：

  • CSS Writing Modes 全系列 vendor prefixes
  • 繁體中文　2026　年　數字自動直立
  • Ruby 注音：繁體ㄈㄢˊ ㄊㄧˇ、豎排ㄕㄨˋ ㄆㄞˊ
  • 圖片、表格自動轉為橫排
  • 章節分頁控制
  • 兩端對齊段落

第二節　相關研究

針對電子書豎排文字的研究可追溯至 EPUB 2 時代，但受限於當時的 CSS 標準，實現方式極為繁瑣且兼容性差。EPUB 3 的推出帶來了標準化的書寫模式支援，使得情況大為改善。

以下是主要閱讀器對豎排文字的支援情況：

閱讀器 支援程度 備註
Apple Books ✅ 完整支援 使用 -webkit-writing-mode
Google Play Books ✅ 完整支援 支援混合格式
Calibre ✅ 完整支援 推薦使用
Thorium Reader ✅ 完整支援 Readium CSS 標準
Amazon Kindle ⚠️ 有限支援 需透過 Calibre 轉換
KOReader ⚠️ 部分支援 可能有渲染問題

由此可見，採用標準化的 CSS 書寫模式，配合完善的 vendor prefixes，是目前最廣泛兼容的做法。

提示：不同的閱讀器對 text-orientation 的預設值可能不同。如需嚴格控制，可明確設定為 upright（所有字符直立）或 mixed（標點符號旋轉）。
